# Spring Festival Magic Calculator

一个模仿手机计算器外观的“时间预言”小魔术页面。  
表面上是普通加法，最后却能精准“算出当前时间”。

## 项目亮点

- 高拟真计算器 UI：黑底、大号数字、圆形按键，舞台感很强。
- 节奏设计完整：输入、求和、停顿、自动补数、最终揭晓。
- 交互细节到位：按键高亮、运算符状态、锁定防误触、字体自适应。
- 纯前端实现：一个 `HTML` 文件即可运行，无需后端。

## 快速开始

1. 克隆仓库
2. 直接用浏览器打开 `春晚魔术.html`
3. 按下面流程演示

## 演示流程（给表演者）

1. 输入第一个数字 `A`
2. 按 `+`
3. 输入第二个数字 `B`
4. 按 `=`，得到 `S = A + B`
5. 再按一次 `+`，页面会锁定 3 秒并自动逐位输入一个数字
6. 最后按 `=`，结果会显示当前时间编码值 `T`（格式：`月日时分`，如 `02162345`）

## 原理说明（代码逻辑）

- 第一次 `+`：保存 `A` 到 `runningTotal`，等待输入下一个数字
- 第一次 `=`：计算 `S = A + B`
- 第二次 `+`：计算 `diff = T - S`，并通过动画逐位输入 `diff`
- 第二次 `=`：计算 `S + diff`，结果恒等于 `T`

也就是：

`(A + B) + (T - (A + B)) = T`

## 关键函数

- `getMagicTime()`：生成当前时间编码（`月日时分`）
- `handlePlus()`：控制两次 `+` 的不同阶段逻辑
- `handleEquals()`：控制两次 `=` 的计算逻辑
- `animateMagicTyping()`：自动逐位输入“神秘数字”
- `setLock()`：动画期间锁定按键，防止误操作
- `updateDisplay()`：显示数字并按位数自适应字号

## 使用建议

- 演示前确认系统时间正确。
- 建议前两次输入不要太大，避免中间出现负数影响观感。
- 推荐全屏展示，舞台效果更好。
